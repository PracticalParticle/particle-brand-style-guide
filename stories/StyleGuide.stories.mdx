# Particle Design System — Style Guide

A single source of truth for components, tokens, and patterns used across Particle Crypto Security LTD and BloxChain Protocol.

---

## 1. Foundation

### Design Tokens

Use theme-aware tokens instead of raw colors or spacing values.

| Category | Tailwind classes | Usage |
|----------|------------------|--------|
| **Backgrounds** | `bg-primary`, `bg-secondary`, `bg-tertiary` | Page, cards, subtle sections |
| **Text** | `text-text-primary`, `text-text-secondary`, `text-text-tertiary`, `text-text-inverse` | Body, labels, placeholders, text on dark |
| **Borders** | `border-default`, `border-hover`, `border-focus` | Default, hover, and focus borders |
| **Brand** | `bg-primary`, `text-primary`, `accent-primary`, `ring-primary` | CTAs, links, focus rings, checkbox/radio accent |
| **Semantic** | `success`, `error`, `warning`, `info` | Status, validation, alerts |

### Typography

- **Font**: Inter (sans), JetBrains Mono (mono).
- **Scale**: `text-xs` … `text-5xl` (see Design Tokens for full scale).
- Prefer semantic text colors: `text-text-primary` for body, `text-text-secondary` for supporting text.

### Spacing & layout

- Use the spacing scale (`p-4`, `gap-3`, `space-y-2`, etc.) for consistency.
- Form controls: `space-y-1.5` between label/helper and `gap-2` between control and label.

### Theme (light / dark)

- Toggle via Storybook toolbar or `dark` class on a parent.
- All tokens switch with theme; avoid hard-coded hex values.

## 3. Form controls pattern

All form controls in this system follow the same structure:

- **Container**: `flex flex-col space-y-1.5`
- **Row**: control + label with `flex items-start gap-2` (or `items-center` for Switch)
- **Label**: `text-sm font-medium text-text-primary` (and `cursor-pointer` where it toggles the control)
- **Helper text**: `text-sm text-text-tertiary ml-6` (or equivalent)
- **Error**: `text-sm text-error ml-6` (and `border-error` / `focus:ring-error` on the control)

Checkbox and Radio use:

- Border: `border-2 border-default`, hover: `hover:border-hover`
- Checked/accent: `accent-primary`
- Focus: `focus:outline-none focus:ring-2 focus:ring-primary focus:ring-offset-0` (and `focus-visible:*` for accessibility)
- Disabled: `disabled:opacity-50 disabled:cursor-not-allowed disabled:hover:border-default`

## 4. Quick reference

### Common patterns

```tsx
// Page background
<div className="min-h-screen bg-primary">  // or bg-secondary for cards

// Card with border
<Card className="border border-default/25">

// Form label + helper
<label className="text-sm font-medium text-text-primary">Label</label>
<p className="text-sm text-text-tertiary">Helper text</p>

// Error state
<p className="text-sm text-error">Error message</p>
<input className="border-error focus:ring-error" />
```

## 5. Accessibility

- Use semantic HTML and `aria-*` where needed (e.g. `aria-invalid`, `aria-label` for icon-only controls).
- Focus: use `focus-visible:ring-*` so keyboard focus is visible and mouse focus can be minimal.
- Contrast: theme tokens are chosen for WCAG 2.1 AA; prefer `text-text-primary` on `bg-primary`/`bg-secondary` for body text.
