# Brand identity

This section is the **single source for brand identity** used across:

- **Product (SaaS)** — app headers, footers, favicons
- **Company website** — marketing pages, link previews
- **Social media** — profile images, Open Graph, Twitter cards
- **Other channels** — email, presentations, print

Use the assets and specs here for consistency everywhere.

## What's in this section

| Asset | Purpose |
|-------|--------|
| [Logo](/story/brand-logo--logo-collection) | Variants, sizes, shapes, download (SVG). Use for all channels. |
| [Social and Open Graph](/story/brand-social-and-open-graph--default) | Dimensions and examples for link previews (e.g. 1200×630). |
| [Favicon](/story/brand-favicon--default) | Browser tab and bookmark icons (16, 32, 48, 64 px). |

## Guidelines and tokens

- **Narrative and usage:** See the Brand Guidelines document (`docs/BRAND_GUIDELINES.md`) in the repo for voice, do's and don'ts, and logo usage.
- **Colors and typography:** Use [Design System / Design Tokens](./?path=/story/design-system-design-tokens--colors) for palette and type scale; brand assets use the same tokens.

Do not duplicate token documentation here — link to the design system for colors, spacing, and typography.
